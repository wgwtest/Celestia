# Celestia MVC Step8 Cross-Process 3D View and Scene Protocol Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` or `superpowers:executing-plans` to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将现有 OpenGL 3D View 从 in-process 主路径推进到独立 View 进程，并通过可序列化 scene protocol 与 Model / Controller runtime 对接。

**Architecture:** Step8 复用 Step7 的 `RuntimeSession`、`HostProcess`、`RuntimeEnvelope` 和 supervisor 生命周期，不重新发明进程监管。Model 只输出可序列化的 scene/view state 和 resource reference；View3D host 独立持有 SDL window、OpenGL context、Renderer、GPU resource cache 和 render loop。Controller 继续只产生命令和输入事件，不直接访问 Model 或 View 内部对象。

**Tech Stack:** C++17, CMake, doctest, SDL2, OpenGL, existing Celestia renderer/resource loaders, `celestia_runtime`, Step7 stdio-pipe transport.

---

## 1. 阶段定位

Step7 已经证明：

```text
M / C / V 三 host 可以由 RuntimeSession 启动和监管。
stdio-pipe 可以作为本机长期 IPC 主路径。
2D debug view 可以在 multi-process serve 路径下持续接收 view.frame。
现有 OpenGL 3D 仍然只在 in-process 路径中运行。
```

Step8 解决的是：

```text
OpenGL 3D View 能不能真正作为独立 View 进程运行。
Model 能不能不暴露 CelestiaCore / Simulation / Universe / Renderer 指针给 View。
View3D 能不能通过 scene protocol 和本地资源引用渲染最小真实场景。
View3D 进程能不能拥有自己的窗口、OpenGL context 和事件循环。
```

Step8 不解决：

```text
DLL hot reload。
View ABI 稳定化。
shared memory 数据面。
跨机器部署。
移除全部历史兼容头。
完整生产级 renderer 重写。
```

## 2. 目标能力

Step8 完成后可以说：

```text
3D View 已具备跨进程运行的最小闭环。
Model -> View3D 的数据边界变成 scene protocol，而不是 C++ 指针边界。
View3D 进程自己管理 SDL window / OpenGL context / render loop / GPU resource cache。
Controller 输入事件可以经 IPC 返回 Controller，再驱动 Model 更新。
现有 in-process 3D 路径仍可作为兼容主路径保留。
```

Step8 完成后仍不能说：

```text
任意 View 已经热插拔。
View 插件 ABI 已经稳定。
所有 2D / 3D View 已经统一 runtime 装配。
所有高频渲染数据已经走 shared memory。
所有历史兼容头已移除。
```

## 5. 目标运行拓扑

```text
celestia-sdl.exe
  RuntimeSession
    celestia-controller-host --serve
      receives view.input
      sends model commands

    celestia-model-host --serve
      owns simulation-facing scene extraction
      sends scene.frame

    celestia-view3d-host --serve
      owns SDL window
      owns OpenGL context
      owns renderer/resource cache
      receives scene.frame
      sends view.input
```

消息方向：

```text
launcher -> all:
  runtime.hello
  runtime.start
  runtime.shutdown

view3d -> controller:
  view.input

controller -> model:
  model.step
  model.setTime
  model.setObserver
  model.selectObject

model -> view3d:
  scene.frame
  scene.delta

view3d -> launcher:
  view.ready3d
  runtime.heartbeat
  runtime.stopped
```

## 9. 风险和缓解

| 风险 | 表现 | 缓解 |
| --- | --- | --- |
| 直接复用 in-process renderer 假定 | View3D host 编译或运行时需要 CelestiaCore 指针 | scene protocol 测试禁止 renderer/OpenGL 指针进入 Model payload |
| OpenGL context 生命周期错误 | 窗口黑屏、崩溃、资源泄漏 | View3D loop 单独封装 init/render/shutdown，并用 smoke 截图验收 |
| 资源路径不稳定 | View3D 找不到纹理、mesh、星表 | ResourceRef 只用 content root + relative path，启动时打印 content root |
| IPC 高频帧阻塞 | render loop 卡顿 | Step8 只做最小 scene.frame；Step10 再做 shared memory/data plane |
| 误称插件化完成 | 3D host 能跑但不能热插拔 | 文档明确 Step9 才处理 View plugin ABI |
