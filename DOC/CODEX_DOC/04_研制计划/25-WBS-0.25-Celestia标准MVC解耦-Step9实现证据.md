```text

  celestia_view_plugin_abi_version
  celestia_view_plugin_id
  celestia_create_view_provider
  celestia_destroy_view_provider

Boundary rule:
  The ABI exposes an opaque CelestiaViewProvider handle and C function
  pointers only. It does not expose STL types, C++ exceptions, renderer
  classes, Model objects, Controller objects, SDL types, or OpenGL types.
```

## Current Step9 Boundary

```text
Can say:
  - View discovery is available through a manifest-backed registry.
  - The first stable View plugin ABI exists and is C-compatible.
  - Built-in Debug2D and OpenGL3D views are represented as plugin manifests.
  - Runtime can perform an ordered View switch from Debug2D to OpenGL3D.
  - The switch keeps Model and Controller hosts alive while replacing View.
  - The OpenGL3D View still owns its SDL window, OpenGL context, render loop,
    and input event collection.

Cannot yet say:
  - arbitrary third-party View plugins are a frozen long-term ABI contract.
  - hot-swap is lossless within a rendered frame.
  - View plugins can be replaced at any moment without an ordered stop/start.
  - the high-frequency data plane uses shared memory or another Step10
    transport.
  - all historical compatibility headers and old include paths are removed;
    this belongs to Step11.
```
