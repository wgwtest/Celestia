## Current Step10 Boundary

```text
Can say:
  - Runtime assembly config can select session timing, transport, data-plane
    kind, View id, View switch fields, and trace output.
  - RuntimeSession no longer directly depends on one concrete HostProcess
    communication path; it talks through RuntimeTransport.
  - stdio-pipe and Windows local-socket/named-pipe transports both run real
    Model / Controller / View host lifecycles.
  - Debug2D and OpenGL3D can both be launched from runtime config files.
  - The ordered Debug2D -> OpenGL3D switch works through the same config path.
  - A DataPlaneRef contract, in-process data plane, and minimal Windows shared
    memory data plane exist and are covered by tests.

Cannot yet say:
  - every scene.frame or render resource payload is automatically moved through
    shared memory by default.
  - local-socket is a cross-machine transport; this implementation is a
    same-machine Windows named-pipe path.
  - tcp-socket or network deployment is implemented.
  - arbitrary third-party transport plugins are a frozen public extension
    ecosystem.
  - all migration compatibility headers are removed; this remains Step11.
```
