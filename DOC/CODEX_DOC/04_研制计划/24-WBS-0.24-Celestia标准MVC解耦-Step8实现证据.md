# Celestia MVC Step8 Implementation Evidence

Date: 2026-06-25

## Implemented Scope

```text
scene.frame protocol:
  src/celruntime/protocol/sceneprotocol.h
  src/celruntime/protocol/sceneprotocol.cpp

model scene extraction:
  src/celruntime/model/sceneextractor.h
  src/celruntime/model/sceneextractor.cpp
  ModelService supports model.step/model.requestSnapshot with view=celestia.view3d.opengl.
  ModelService accepts model.setViewInput and reflects input state in the next scene.frame.

view.input protocol:
  src/celruntime/protocol/viewinput.h
  src/celruntime/protocol/viewinput.cpp
  View3D SDL events are encoded as renderer-free view.input envelopes.
  ControllerService converts view.input into model.setViewInput or runtime.shutdown.

independent View3D host:
  src/celruntime/process/view3dhostmain.cpp
  src/celruntime/process/view3dhostloop.*
  src/celruntime/view3d/view3dhost.*
  src/celruntime/view3d/view3dloop.*

runtime routing:
  RuntimeSession selects celestia-view3d-host for celestia.view3d.opengl.
  2D debug route continues to use celestia-view-host and view.frame.
  3D route uses scene.frame and waits for view.frameRendered.
  RuntimeSession drains pending view.input events through Controller -> Model.
```

## Current Step8 Boundary

```text
Can say:
  - 3D View has a minimal independent process path.
  - Model -> View3D uses a serializable scene.frame protocol.
  - View3D -> Controller -> Model uses a serializable view.input command path.
  - View3D host owns its SDL window, OpenGL context, render loop, and scene cache.
  - Existing 2D debug multi-process route remains compatible.

Cannot yet say:
  - full historical Celestia renderer parity is reached in the View3D host.
  - View plugin ABI/hot reload is complete; this belongs to Step9.
  - shared-memory/high-frequency data plane is complete; this belongs to Step10.
  - all compatibility headers are removed; this belongs to Step11.
```
