# Celestia MVC Step7 Long-Running IPC and Process Supervisor Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` or `superpowers:executing-plans` to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将 Step6 的三进程 `--serve` smoke 升级为可长期运行、可监控、可关闭、可持续传递 MVC 消息的本机 M / C / V runtime。

**Architecture:** 复用 Step6 的 `RuntimeEnvelope` / `ViewFrame` / lifecycle 协议，不重新定义消息语义。第一轮把当前临时文件 + 脚本式 smoke 编排替换为真实 live stdio pipe IPC，由 `ProcessSupervisor` 管理三个 host 的生命周期、消息路由、heartbeat、timeout 和 shutdown。OpenGL 3D View 跨进程渲染、DLL hot reload、shared memory 数据面不放入 Step7。

**Tech Stack:** C++17, CMake, doctest, existing `celestia_runtime`, `RuntimeEnvelope`, `FramedMessage`, persistent stdio pipe IPC, Windows `CreateProcess` / anonymous pipe first implementation, SDL launcher smoke, Visual Studio BuildTools CMake/CTest.

---

## 1. Step7 定位

Step6 已经完成：

```text
Runtime Protocol v1
Framed stdio message encoding
M / C / V host --serve lifecycle loop
ModelService / ControllerService / ViewService 最小长期语义
ProcessSupervisor 三进程 smoke
SDL --mvc-mode=multi-process --serve smoke 入口
```

Step6 仍然是 smoke 级闭环：

```text
ProcessSupervisor 主要通过临时输入 / 输出文件和命令脚本驱动 host。
host 进入长期 --serve loop，但 launcher 并未持有真正持续的双向 IPC channel。
Controller -> Model -> View 的持续消息流还不是常驻路由器。
heartbeat / timeout / host 异常退出还没有形成 supervisor 级策略。
```

Step7 的核心是把“能证明三进程可以跑通”升级成“本机三进程 runtime 可以长期在线运行”。

## 2. 统一口径

从本计划开始，阶段口径统一为：

```text
Step7: 长期 IPC + ProcessSupervisor runtime 化
Step8: OpenGL 3D View 跨进程渲染与 GPU 资源边界
Step9: View 插件 ABI / DLL hot reload
Step10: shared memory 数据面与远程 transport
```

这表示 Step7 优先做 Transport / Supervisor / RuntimeSession。之前讨论中的 OpenGL 3D View 进程化候选线顺延为 Step8，避免在长期 IPC 尚未稳定时同时处理 OpenGL context、窗口、GPU 资源和场景同步。

## 3. Step7 验收口径

Step7 完成后可以说：

```text
Celestia 已具备本机长期运行 M / C / V 三进程 runtime。
ProcessSupervisor 不再依赖临时文件 / 脚本式 smoke 来完成主路径 IPC。
Launcher 可以通过 live stdio pipe 持续收发 RuntimeEnvelope。
View -> Controller -> Model -> View 的最小消息流可以连续运行多个 tick。
Supervisor 可以检测 ready / heartbeat / stopped / error / timeout。
SDL launcher 可以启动 multi-process runtime，并在 duration-ms 到达后正常 shutdown 三个 host。
```

```text

任意 2D / 3D View 已经 DLL 热插拔。

M / C / V 已经可以跨机器部署。
真实 Celestia SimulationBackend 已完整迁入独立 Model 进程。
```

```text
  runtime.ready
  runtime.heartbeat

```

```cpp
namespace celestia::runtime::process
{

struct HostProcessOptions
{
    std::filesystem::path executable;
    std::vector<std::string> arguments;
    std::filesystem::path workingDirectory;
};

class HostProcess
{
public:
    explicit HostProcess(HostProcessOptions);
    ~HostProcess();

    HostProcess(const HostProcess&) = delete;
    HostProcess& operator=(const HostProcess&) = delete;

    bool start();
    bool send(const protocol::RuntimeEnvelope&);
    std::optional<protocol::RuntimeEnvelope> receive(std::chrono::milliseconds timeout);
    std::optional<int> wait(std::chrono::milliseconds timeout);
    bool terminate();
    bool running() const;

private:
    class Impl;
    std::unique_ptr<Impl> impl_;
};

} // namespace celestia::runtime::process
```

- [ ] **Step 2: 实现 Windows live pipe**

Windows 第一实现使用：

```text
CreatePipe
SetHandleInformation(..., HANDLE_FLAG_INHERIT, 0)
CreateProcessW
ReadFile
WriteFile
PeekNamedPipe
WaitForSingleObject
TerminateProcess
CloseHandle
```

实现规则：

```text
child stdin: parent write pipe -> child STD_INPUT_HANDLE
child stdout: child STD_OUTPUT_HANDLE -> parent read pipe
child stderr: 可以继承父进程 stderr，Step7 第一轮不解析 stderr
receive(timeout): 轮询 stdout pipe，使用 FramedMessageReader 解码 RuntimeEnvelope
send(envelope): 使用 encodeFrame(envelope)，写入 child stdin pipe
destructor: 若进程仍运行，先尝试 terminate，再关闭所有 HANDLE
```

```cpp
namespace celestia::runtime::process
{

struct RuntimeSessionOptions
{
    std::filesystem::path runtimeHostDirectory;
    std::string viewId{ "celestia.view2d.debug" };
    std::string sessionId{ "default" };
    int durationMilliseconds{ 1000 };
    int tickMilliseconds{ 125 };
    int hostReadyTimeoutMilliseconds{ 3000 };
    int shutdownTimeoutMilliseconds{ 3000 };
};

struct RuntimeSessionResult
{
    bool success{ false };
    bool modelReady{ false };
    bool controllerReady{ false };
    bool viewReady{ false };
    bool modelStopped{ false };
    bool controllerStopped{ false };
    bool viewStopped{ false };
    int modelExitCode{ -1 };
    int controllerExitCode{ -1 };
    int viewExitCode{ -1 };
    int tickCount{ 0 };
    int viewFrameCount{ 0 };
    std::string log;
};

class RuntimeSession
{
public:
    explicit RuntimeSession(RuntimeSessionOptions);
    RuntimeSessionResult run();

private:
    RuntimeSessionOptions options_;
};

} // namespace celestia::runtime::process
```

- [ ] **Step 3: 实现最小路由循环**

第一轮路由规则：

```text
startup:
  launcher -> each host: runtime.hello
  expect each host -> launcher: runtime.ready
  launcher -> each host: runtime.start

tick:
  launcher -> controller: controller.tick
  controller -> model: model.step or model.requestSnapshot
```

## 8. 风险与缓解

| 风险 | 表现 | 缓解 |
| --- | --- | --- |
| 继续使用临时文件 smoke | 不能证明长期 IPC | Task 1/2 必须证明 live pipe 收发 |
| live pipe 读写阻塞 | SDL 或测试卡死 | receive(timeout)、shutdown timeout、terminate 策略必须有测试 |
| Supervisor 变成业务层 | Launcher 侵入 Model / Controller / View 语义 | Supervisor 只路由 RuntimeEnvelope，不读写 Model 内部对象 |
| 过早做 3D 跨进程 | IPC、窗口、GPU、scene state 同时失败 | Step7 继续用 Debug 2D / headless View 验收 |
| Windows-only 实现被误认为跨平台 | Linux/macOS 构建误判完成 | 非 Windows 显式 unsupported，并在文档标注 |
| 长期运行遗留 host 进程 | 测试结束后进程残留 | HostProcess destructor / RuntimeSession shutdown / terminate tests |
| 输出依赖本机数据目录 | 3D smoke 出现 Cannot read star database | 验证前确保 `run-full\data\stars.dat` 存在 |
