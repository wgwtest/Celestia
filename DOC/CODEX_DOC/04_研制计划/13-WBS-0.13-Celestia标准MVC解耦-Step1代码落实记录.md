# WBS-0.13 Celestia 标准 MVC 解耦 Step 1 代码落实记录

## 目标边界

Step 1 是 Celestia 源码内的 MVC 代码级解耦。它不等同于文档归类，也不包含 Planet_SIM 迁移。只有在 Celestia 本仓库完成代码解耦、构建出可运行程序并通过全量测试后，才允许进入 Step 2 的本仓库内更彻底 MVC 解耦。

## 已落地代码切片

| 切片 | 代码产物 | 结果 |
| --- | --- | --- |
| `Simulation` 控制层收缩 | 删除 `render` / `pickObject`，移除 `Renderer` / `RenderFlags` 头文件暴露 | 控制层不再转发 View 操作 |
| `Universe` 模型层收缩 | 删除 `GeometryManager` 构造依赖和 `pick*` 实现 | 模型聚合不再持有拾取所需 View 资源 |
| 拾取适配层 | 新增 `SelectionPicker`、`DeepSkyObjectPicker` | 拾取逻辑集中在 View Adapter |
| DSO 渲染策略 | 新增 `DeepSkyObjectRenderPolicy` | DeepSkyObject 模型头不再暴露 `RenderFlags` / `RenderLabels` |
| Body 位置几何投影 | 新增 `BodyLocationGeometryProjector` | `BodyFeaturesManager` 不再直接暴露 `GeometryManager` 计算接口 |
| Body 渲染资产 | 新增 `BodyRenderAssets` | `Body` 模型头不再暴露主几何、主表面、alternate surface、rings 纹理、`TextureHandle` 和 `meshmanager.h` |
| Star 渲染资产 | 新增 `StarRenderAssets` | `Star` / `StarDetails` 模型头不再暴露纹理、网格和 `meshmanager.h` |
| Nebula 渲染资产 | 新增 `NebulaRenderAssets` | `Nebula` 模型头不再暴露 `GeometryHandle` |
| 应用编排 | `CelestiaCore` 直接编排 `Renderer` 与 View Adapter | Application Shell 承担编排，不污染 Model / Controller |

## 边界测试

新增 `test/unit/mvc_boundary_test.cpp`，当前覆盖 6 个边界用例：

```text
Simulation controller header does not expose view adapter API
Universe model header does not expose picking or view resources
Body feature manager header does not expose geometry adapter API
Body model header does not expose primary render assets
Star model header does not expose render assets
Deep sky model headers do not expose render mask API
```

这些测试用于防止后续修改把 `Renderer`、`RenderFlags`、`GeometryManager`、`GeometryHandle`、`TextureHandle`、主表面、alternate surface、ring texture 或拾取接口重新放回模型/控制层头文件。

## 构建与测试记录

基线 Release 配置：

```text
build-mvc-baseline-rel
cmake --build build-mvc-baseline-rel --parallel 8: passed
ctest --output-on-failure: passed, 42/42
```

SDL 可运行程序配置：

```text
build-mvc-sdl-rel
cmake --build build-mvc-sdl-rel --target celestia-sdl --parallel 8: passed
cmake --build build-mvc-sdl-rel --parallel 8: passed
ctest --output-on-failure: passed, 42/42
```

可执行程序：

```text
build-mvc-sdl-rel/src/celestia/sdl/celestia-sdl.exe
```

运行验证：已在 `build-mvc-sdl-rel/run-minimal` 准备最小 `celestia.cfg` 和 `test/data/nearstars.stc`，设置 `CELESTIA_DATA_DIR` 后启动 `celestia-sdl.exe`。进程越过配置和数据加载并进入 SDL 运行循环；验证后已结束进程。

## 说明

原生 Win32 前端在当前 MSVC/fmt 组合下存在既有 `winmain.cpp` 编译问题，本次没有把它作为 Step 1 验收前端。Step 1 的可运行程序验收使用 SDL 前端完成；SDL2 已安装到本机 vcpkg 的 `x64-windows` triplet。
