# Celestia MVC Step12 Real SceneFrame Contract Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:executing-plans` to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking. This Step12 plan is the contract gate for Step13 through Step18; do not start Step13 Model backend work before this plan's acceptance evidence is recorded.

**Goal:** 为后续真实 Model Backend、真实 SceneExtractor、跨进程 View3D 渲染和 Controller 交互建立稳定的 `scene.frame vNext` 契约、fixture、验证测试和 Model Lock Policy。

**Architecture:** Step12 采用 contract-first 方式推进。它只定义 Model 输出给 View 的稳定数据边界，并增加可自动验证的协议样例和断言；它不加载真实 Celestia Universe，不迁移历史 Renderer，也不扩大截图回归矩阵。当前实现已有 `src/celruntime/protocol/sceneprotocol.*`、`SceneFrame`、`SceneExtractor` 和 Step8 测试，Step12 在这些基础上演进 vNext 字段和验证规则。

**Tech Stack:** C++17, CMake, doctest, PowerShell, existing `sceneprotocol` flat key-value serializer, `tools/mvc` dependency scans, `tools/regression` compatibility harness.

**Implementation status (2026-06-27):** Step12 contract implementation is implemented on branch `codex/celestia-mvc-step12-scene-frame` at checkpoint commit `0fd93ad`. The branch has added `scene.frame vNext` DTO fields, validation helpers, fixtures, and Step12 unit coverage. Verified: SDL `unit` build, SDL full CTest `166/166`, baseline `unit` build, baseline full CTest `166/166`, MVC dependency scan, MVC CMake target scan, and Full screenshot compatibility regression `2026-06-27-173305`.

## 1. 需求判断

单独写 Step12 详细计划是合理且必要的。

原因：

```text
Step12 是后续 Step13-17 的共同契约门闩。
Step13 真实 Model Backend 需要知道输出哪些字段。
Step14 真实 SceneExtractor 需要知道如何投影真实对象。
Step15 View3D 需要知道只消费哪些数据。
Step16 Controller 需要知道哪些状态变化会进入下一帧。
Step17 ResourceRef / data-plane 需要知道资源字段的稳定形态。
```

如果没有单独 Step12 计划，后续最容易出现的问题是：

```text
Model 会为了 View 需求临时加字段，破坏 Model Lock Policy。
View3D 会直接依赖 Model 内部对象，绕回未解耦结构。
Controller 命令和 scene.frame 状态字段不匹配。
ResourceRef 规则后补，导致前面字段重做。
回归测试只能截图，缺少协议级失败定位能力。
```

Step12 的正确产出是：

```text
一份正式 scene.frame vNext 设计说明。
一组可机器验证的 frame fixtures。
一批协议和边界测试。
一条明确 Model Lock Policy。
一份 Step12 机测记录和可说/不可说边界。
```

## 5. scene.frame vNext 最小契约

Step12 的 vNext 不是把所有未来能力一次实现，而是冻结后续可扩展的稳定骨架。

### 5.1 顶层字段

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `protocolVersion` | int | 是 | Step12 固定为 `2` |
| `sessionId` | string | 是 | runtime session id |
| `sequence` | uint64 | 是 | 单调递增 frame 序号 |
| `time` | TimeState | 是 | Celestia simulation time |
| `camera` | CameraState | 是 | View 相机状态 |
| `observer` | ObserverState | 是 | Celestia observer 只读快照 |
| `renderSettings` | RenderSettings | 是 | 可见层与基础渲染设置 |
| `resources` | ResourceRef[] | 否 | frame 使用的资源引用 |
| `bodies` | BodyRenderState[] | 否 | 太阳系天体投影 |
| `stars` | StarRenderState[] | 否 | 恒星投影 |
| `deepSkyObjects` | DeepSkyRenderState[] | 否 | DSO 投影 |
| `orbits` | OrbitRenderState[] | 否 | 轨道线投影 |
| `labels` | LabelRenderState[] | 否 | 标签投影 |
| `selection` | SelectionState | 否 | 当前选择 |

### 5.2 时间字段

```cpp
struct TimeState
{
    double julianDayTdb{ 0.0 };
    double secondsSinceJ2000{ 0.0 };
    double timeScale{ 1.0 };
    bool paused{ false };
};
```

规则：

```text
Step12 可以继续保留 legacy simulationTime，用于兼容 Step8 tests。
vNext 正式字段必须包含 time.julianDayTdb。
Step13 实现真实 backend 时必须把真实 Celestia 时间映射到 time.julianDayTdb。
View3D 不得根据本机时钟自行推导 Celestia 时间。
```

### 5.3 对象 id 规则

稳定 object id 使用逻辑 id，不使用指针地址、数组下标或本机资源句柄。

建议格式：

```text
celestia:body:Sol/Earth
celestia:body:Sol/Earth/Moon
celestia:star:Sol
celestia:dso:M31
celestia:orbit:Sol/Earth
```

验证规则：

```text
object id must not be empty.
object id must not contain process-local pointer syntax.
object id must not contain a Windows drive prefix.
object id should be stable across frames for the same Celestia object.
```

### 5.4 ResourceRef 规则

Step12 推荐将 `ResourceRef` 扩展为：

```cpp
struct ResourceRef
{
    std::string id;
    std::string kind;
    std::string package;
    std::string relativePath;
    std::string contentHash;
    std::string dataPlaneKey;
    bool required{ false };
};
```

规则：

```text
relativePath must be relative to the configured content root.
relativePath must not contain ":\\".
relativePath must not start with "/" or "\\".
relativePath must not contain ".." path traversal.
dataPlaneKey may be empty in Step12.
GPU handles such as GLuint, texture pointer, mesh pointer, Renderer pointer are forbidden.
```

### 5.5 LabelRenderState

当前 `labels` 是 `std::vector<std::string>`。Step12 应升级为结构化 label：

```cpp
struct LabelRenderState
{
    std::string targetObjectId;
    std::string text;
    std::string kind;
    bool visible{ true };
};
```

兼容策略：

```text
Step12 可以在 serializer 中同时支持旧 labelN 和新 labelN.text。
Step15 以后 View3D 只应依赖结构化 label。
```

### 5.6 DeepSkyRenderState

Step12 只定义最小 DSO 结构，不要求 Step12 输出真实 DSO 数据。

```cpp
struct DeepSkyRenderState
{
    std::string objectId;
    std::string name;
    std::string dsoType;
    std::array<double, 3> positionKm{ 0.0, 0.0, 0.0 };
    double apparentMagnitude{ 0.0 };
    bool visible{ true };
    ResourceRef catalogResource;
};
```

## 6. validation helper 契约

Step12 应在 `sceneprotocol` 中增加轻量 validation，不引入重型 schema 引擎。

建议接口：

```cpp
struct SceneFrameValidationIssue
{
    std::string field;
    std::string message;
};

std::vector<SceneFrameValidationIssue> validateSceneFrame(const SceneFrame&);
bool isValidSceneFrame(const SceneFrame&);
```

最低检查：

```text
protocolVersion == SceneFrameProtocolVersion
sessionId is not empty
sequence is present
time.julianDayTdb is finite
camera.fov > 0
camera.nearPlane > 0
camera.farPlane > camera.nearPlane
observer.frame is not empty
object ids are not empty when objects are present
ResourceRef relativePath is relative and safe
payload text does not contain forbidden process-local tokens
```

禁止 token：

```text
Simulation*
Universe*
CelestiaCore*
Renderer*
Texture*
Mesh*
GLuint
0x
:\
```
